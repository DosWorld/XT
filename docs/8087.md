## 1. Overview

The Intel 8087 (1980) is a floating-point coprocessor designed to work alongside the 8086 and 8088 CPUs. It adds hardware support for IEEE 754-compatible floating-point arithmetic, removing the need for slow software floating-point libraries.

The 8087 monitors the CPU bus. When it sees an ESCape opcode (opcodes `D8`–`DF`) it executes the instruction independently, while the main CPU either waits (after a `FWAIT` / `9B` prefix) or continues executing the next instruction. Synchronisation between CPU and FPU is the programmer's responsibility.

Subsequent generations: **8087** (8086/8088), **80287** (286), **80387** (386). The 80387 added `FSIN`, `FCOS` and other instructions absent from the 8087. Unless noted, this manual covers the **8087** only.

## 2. Architecture

### 2.1 Register Stack

The 8087 has eight 80-bit floating-point registers organised as a **circular stack**. They are not addressed by fixed number; instead the hardware maintains a **TOP** pointer (3 bits, field in the status word, bits 13–11) that indicates which physical register is currently `ST(0)`.

```
ST(0)  — top of stack  (most recent push)
ST(1)  — one below top
ST(2)
...
ST(7)  — bottom of stack
```

`ST(i)` maps to physical register `(TOP + i) mod 8`.

**Push** (`FLD`, `FILD`, `FBLD`) decrements TOP by 1 and writes to the new `ST(0)`.  
**Pop** (`FSTP`, `FISTP`, `FBSTP` and the `P`-suffix arithmetic instructions) reads `ST(0)` then increments TOP by 1, marking the old slot as **Empty** in the tag word.

Stack overflow (push onto a non-empty slot) and stack underflow (pop from an empty slot) are flagged in the status word.

Each physical register holds an **80-bit extended-real** value internally, regardless of the data type used to load it.

### 2.2 Control Word

A 16-bit register that controls rounding, precision, and exception masking.

```
Bit  15–12  Reserved
Bit  11–10  RC  Rounding Control
Bit   9– 8  PC  Precision Control
Bit   7     X   (reserved)
Bit   6     PM  Precision exception Mask
Bit   5     UM  Underflow exception Mask
Bit   4     OM  Overflow exception Mask
Bit   3     ZM  Zero-divide exception Mask
Bit   2     DM  Denormal exception Mask
Bit   1     IM  Invalid-operation exception Mask
Bit   0     (reserved)
```

**RC — Rounding Control (bits 11–10)**

| RC | Mode |
|----|------|
| 00 | Round to nearest (even) — default |
| 01 | Round toward −∞ (floor) |
| 10 | Round toward +∞ (ceiling) |
| 11 | Round toward zero (truncate) |

**PC — Precision Control (bits 9–8)**

| PC | Mantissa bits | Type |
|----|---------------|------|
| 00 | 24 bits | Single (32-bit) |
| 01 | (reserved) | — |
| 10 | 53 bits | Double (64-bit) |
| 11 | 64 bits | Extended (80-bit) — default |

**Default control word after `FNINIT`: `0x037F`**  
All exceptions masked, extended precision, round-to-nearest.

### 2.3 Status Word

A 16-bit register reflecting the state of the FPU after each instruction.

```
Bit  15     B    FPU Busy (same as ES for 8087)
Bit  14     C3   Condition code bit 3
Bit 13–11   TOP  Top-of-stack pointer (0–7)
Bit  10     C2   Condition code bit 2
Bit   9     C1   Condition code bit 1 / stack overflow indicator
Bit   8     C0   Condition code bit 0
Bit   7     ES   Error Summary (any unmasked exception)
Bit   6     SF   Stack Fault
Bit   5     PE   Precision exception
Bit   4     UE   Underflow exception
Bit   3     OE   Overflow exception
Bit   2     ZE   Zero-divide exception
Bit   1     DE   Denormal exception
Bit   0     IE   Invalid-operation exception
```

**Condition code bits after `FCOM`/`FUCOM`:**

| Relation     | C3 | C2 | C0 |
|--------------|----|----|-----|
| ST(0) > src  | 0  | 0  | 0  |
| ST(0) < src  | 0  | 0  | 1  |
| ST(0) = src  | 1  | 0  | 0  |
| Unordered    | 1  | 1  | 1  |

**Condition code bits after `FXAM`:**

| Class    | C3 | C2 | C1 (sign) | C0 |
|----------|----|----|-----------|----|
| Normal   | 0  | 1  | sign      | 0  |
| Zero     | 1  | 0  | sign      | 0  |
| Infinity | 0  | 1  | sign      | 1  |
| NaN      | 0  | 0  | sign      | 1  |
| Empty    | 1  | 0  | 0         | 1  |
| Denormal | 1  | 1  | sign      | 0  |

### 2.4 Tag Word

A 16-bit register with 2 bits per physical register (8 registers × 2 bits = 16 bits).

| Tag | Meaning |
|-----|---------|
| 00  | Valid (non-zero, non-special) |
| 01  | Zero |
| 10  | Special (NaN, infinity, denormal) |
| 11  | Empty |

The tag word is used by `FRSTOR`/`FSAVE` and examined by applications to determine stack depth.

## 3. Data Types

The 8087 natively supports seven data formats:

| Type | Size | Mantissa bits | Range (approx) |
|------|------|---------------|----------------|
| Word integer | 16-bit | — | −32768 to 32767 |
| Short integer | 32-bit | — | −2³¹ to 2³¹−1 |
| Long integer | 64-bit | — | −2⁶³ to 2⁶³−1 |
| Single real | 32-bit | 23+1 | ±1.18×10⁻³⁸ to ±3.40×10³⁸ |
| Double real | 64-bit | 52+1 | ±2.23×10⁻³⁰⁸ to ±1.80×10³⁰⁸ |
| Extended real | 80-bit | 64 | ±3.37×10⁻⁴⁹³² to ±1.18×10⁴⁹³² |
| Packed BCD | 80-bit | 18 decimal digits | ±10¹⁸−1 |

All formats are stored in **little-endian** byte order in memory.

**Single real (32-bit) layout:**
```
Bit 31       Sign
Bits 30–23   Exponent (biased by 127)
Bits 22–0    Mantissa (implicit leading 1)
```

**Double real (64-bit) layout:**
```
Bit 63       Sign
Bits 62–52   Exponent (biased by 1023)
Bits 51–0    Mantissa (implicit leading 1)
```

**Extended real (80-bit) layout:**
```
Bit 79       Sign
Bits 78–64   Exponent (biased by 16383)
Bits 63–0    Mantissa (explicit leading 1 — J-bit)
```
Unlike single and double, the extended format has an explicit integer bit (J-bit = bit 63), making it unnormalised-capable.

**Packed BCD (80-bit) layout:**
```
Byte 9       Sign (bit 7 = 1 for negative, 0 for positive)
Bytes 8–0    18 BCD digits, two per byte (high nibble = tens, low nibble = units)
             Byte 8 holds the two most-significant digits.
```

## 4. Detecting the 8087

Before using 8087 instructions in a program, it is good practice to verify one is present. There is no single-instruction test; detection relies on observing side effects of FPU behaviour.

### Method 1 — Control Word Test (most reliable)

After `FNINIT`, the control word is `0x037F`. Write a known pattern, read it back; if the value survives the FPU has a working control register.

```asm
        FNINIT                  ; initialize without FWAIT
        FSTCW   [cw_save]       ; store control word
        FWAIT                   ; wait for completion
        MOV     AX, [cw_save]
        AND     AX, 0x103F      ; mask reserved bits
        CMP     AX, 0x003F      ; expected lower byte pattern
        JNE     no_fpu
        ; 8087 present
        JMP     fpu_ok
no_fpu:
        ; no FPU
fpu_ok:
```

A simpler variant that checks only the initialization default:

```asm
        FNINIT
        MOV     WORD PTR [cw_save], 0x5A5A   ; fill with sentinel
        FSTCW   [cw_save]
        FWAIT
        MOV     AX, [cw_save]
        CMP     AL, 0x7F                     ; low byte should be 7F after FNINIT
        JNE     no_fpu
```

### Method 2 — Status Word Test

`FNINIT` clears the status word to zero. Check that the stored value reflects this.

```asm
        FNINIT
        MOV     WORD PTR [sw_save], 0xFFFF   ; sentinel
        FSTSW   [sw_save]
        FWAIT
        MOV     AX, [sw_save]
        AND     AX, 0xFF00
        JNZ     no_fpu           ; high byte should be 0 after FNINIT
```

### Method 3 — Tag Word Test (8087/80287 distinction)

On the 8087, `FNINIT` sets all tags to 11 (Empty = `0xFFFF`). Read the tag word to confirm.

```asm
        FNINIT
        MOV     WORD PTR [tw_save], 0x5555
        FSTENV  [env_save]       ; saves 14-byte environment
        FWAIT
        MOV     AX, [env_save+8] ; tag word is at offset 8 in the environment
        CMP     AX, 0xFFFF
        JNE     no_fpu
```

### Method 4 — Interrupt 11h (Equipment List, BIOS)

The BIOS equipment-list word (INT 11h, or read from `0x0040:0x0010`) has bit 1 set when a coprocessor is installed (set by the motherboard sense logic, not software-detectable in all BIOSes).

```asm
        INT     11h              ; get equipment list
        TEST    AX, 0x0002       ; bit 1 = coprocessor installed
        JZ      no_fpu
```

This reflects the hardware jumper/switch setting and is unreliable on machines where the OS has patched the value. Always prefer Method 1.

### Distinguishing 8087 from 80287/80387

The 8087 initialises `ST(x)` tag bits as Empty after `FNINIT`. The 80287 does too but has different infinity semantics. The 80387 added `FUCOM`, `FSIN`, `FCOS` etc.; testing whether those execute without an invalid opcode fault identifies a 387+.

For the purpose of DOS programs targeting PC-XT and PC-AT hardware, simply detecting "FPU present" via Method 1 is sufficient.

## 5. Instruction Reference

All 8087 instructions begin with an ESCape byte (`D8`–`DF`) and are preceded by a `FWAIT` (`9B`) in most assembler output to ensure the CPU and FPU stay in sync.

Notation:
- `ST(0)` — top of stack, also written `ST`
- `ST(i)` — stack register i positions below the top
- `m32` — 32-bit memory operand (single real)
- `m64` — 64-bit memory operand (double real)
- `m80` — 80-bit memory operand (extended real)
- `m16i` — 16-bit integer memory operand
- `m32i` — 32-bit integer memory operand
- `m64i` — 64-bit integer memory operand
- `m80bcd` — 80-bit packed BCD memory operand

### 5.1 Data Transfer

#### FLD — Load Real

Push a value onto the stack.

| Mnemonic | Operand | Encoding | Description |
|----------|---------|----------|-------------|
| `FLD m32` | single real | D8 /0 mem | Push m32 onto stack |
| `FLD m64` | double real | DC /0 mem | Push m64 onto stack |
| `FLD m80` | extended real | DB /5 mem | Push m80 onto stack |
| `FLD ST(i)` | register | D9 C0+i | Push copy of ST(i) onto stack |

```asm
FLD     DWORD PTR [myFloat]     ; push single
FLD     QWORD PTR [myDouble]    ; push double
FLD     TBYTE PTR [myExtended]  ; push extended
FLD     ST(2)                   ; push copy of ST(2)
FLD     ST(0)                   ; duplicate top of stack
```

#### FST — Store Real (no pop)

Copy `ST(0)` to memory or register without popping.

| Mnemonic | Operand | Encoding |
|----------|---------|----------|
| `FST m32` | single real | D9 /2 mem |
| `FST m64` | double real | DD /2 mem |
| `FST ST(i)` | register | DD D0+i |

#### FSTP — Store Real and Pop

Store `ST(0)` then pop the stack.

| Mnemonic | Operand | Encoding |
|----------|---------|----------|
| `FSTP m32` | single real | D9 /3 mem |
| `FSTP m64` | double real | DD /3 mem |
| `FSTP m80` | extended real | DB /7 mem |
| `FSTP ST(i)` | register | DD D8+i |

```asm
FSTP    DWORD PTR [result]      ; store and pop to single
FSTP    QWORD PTR [result]      ; store and pop to double
FSTP    ST(0)                   ; pop and discard
FSTP    ST(1)                   ; store into ST(1) then pop
```

#### FILD — Load Integer

Convert integer to floating-point and push.

| Mnemonic | Operand | Encoding |
|----------|---------|----------|
| `FILD m16i` | word integer | DF /0 mem |
| `FILD m32i` | dword integer | DB /0 mem |
| `FILD m64i` | qword integer | DF /4 mem |

#### FIST — Store Integer (no pop)

Convert `ST(0)` to integer and store.

| Mnemonic | Operand | Encoding |
|----------|---------|----------|
| `FIST m16i` | word integer | DF /2 mem |
| `FIST m32i` | dword integer | DB /2 mem |

Rounding follows the **RC** field of the control word (default: round-to-nearest-even).

#### FISTP — Store Integer and Pop

| Mnemonic | Operand | Encoding |
|----------|---------|----------|
| `FISTP m16i` | word integer | DF /3 mem |
| `FISTP m32i` | dword integer | DB /3 mem |
| `FISTP m64i` | qword integer | DF /7 mem |

#### FBLD — Load Packed BCD

Convert 18-digit packed BCD to floating-point and push.

| Mnemonic | Operand | Encoding |
|----------|---------|----------|
| `FBLD m80bcd` | 80-bit BCD | DF /4 mem |

#### FBSTP — Store Packed BCD and Pop

Convert `ST(0)` to 18-digit packed BCD, store, and pop.

| Mnemonic | Operand | Encoding |
|----------|---------|----------|
| `FBSTP m80bcd` | 80-bit BCD | DF /6 mem |

The result is rounded to the nearest integer before BCD conversion. The sign byte (byte 9) is set to `0x80` for negative, `0x00` for positive.

#### FXCH — Exchange Register

Swap `ST(0)` with `ST(i)`.

| Mnemonic | Operand | Encoding |
|----------|---------|----------|
| `FXCH ST(i)` | register | D9 C8+i |
| `FXCH` | (ST(1) implied) | D9 C9 |

`FXCH` is essentially free on modern FPUs and is the standard way to make `ST(i)` the "active" operand.

### 5.2 Arithmetic

All arithmetic instructions operate on the register stack. The result is written back to one of the operands (in-place). Memory operands are always source-only.

#### FADD — Add

| Mnemonic | Description | Encoding |
|----------|-------------|----------|
| `FADD m32` | ST(0) += m32 | D8 /0 mem |
| `FADD m64` | ST(0) += m64 | DC /0 mem |
| `FADD ST(0), ST(i)` | ST(0) += ST(i) | D8 C0+i |
| `FADD ST(i), ST(0)` | ST(i) += ST(0) | DC C0+i |
| `FADDP ST(i), ST(0)` | ST(i) += ST(0), pop | DE C0+i |
| `FADDP` | ST(1) += ST(0), pop | DE C1 |

#### FSUB — Subtract

| Mnemonic | Description | Encoding |
|----------|-------------|----------|
| `FSUB m32` | ST(0) -= m32 | D8 /4 mem |
| `FSUB m64` | ST(0) -= m64 | DC /4 mem — **Note: DC /5 = FSUBR** |
| `FSUB ST(0), ST(i)` | ST(0) -= ST(i) | D8 E0+i |
| `FSUB ST(i), ST(0)` | ST(i) -= ST(0) | DC E8+i |
| `FSUBP ST(i), ST(0)` | ST(i) -= ST(0), pop | DE E8+i |

**Important — FSUB vs FSUBR encoding in DC/DE group:**

The DC and DE groups swap the subtraction direction relative to D8:

| Group | reg=4 | reg=5 |
|-------|-------|-------|
| D8/DE mem | FSUB (ST(0) − src) | FSUBR (src − ST(0)) |
| DC reg / DE reg | FSUBR (ST(0)−ST(i)) | FSUB (ST(i)−ST(0)) |

This is a documented Intel asymmetry. Assemblers handle it transparently; be aware when reading hand-crafted opcodes.

#### FSUBR — Subtract Reversed

`ST(0) = src - ST(0)` (reversed operand order vs FSUB).

| Mnemonic | Description | Encoding |
|----------|-------------|----------|
| `FSUBR m32` | ST(0) = m32 − ST(0) | D8 /5 mem |
| `FSUBR m64` | ST(0) = m64 − ST(0) | DC /4 mem |
| `FSUBRP ST(i), ST(0)` | ST(i) = ST(0) − ST(i), pop | DE E0+i |

#### FMUL — Multiply

| Mnemonic | Description | Encoding |
|----------|-------------|----------|
| `FMUL m32` | ST(0) *= m32 | D8 /1 mem |
| `FMUL m64` | ST(0) *= m64 | DC /1 mem |
| `FMUL ST(0), ST(i)` | ST(0) *= ST(i) | D8 C8+i |
| `FMUL ST(i), ST(0)` | ST(i) *= ST(0) | DC C8+i |
| `FMULP ST(i), ST(0)` | ST(i) *= ST(0), pop | DE C8+i |

#### FDIV — Divide

| Mnemonic | Description | Encoding |
|----------|-------------|----------|
| `FDIV m32` | ST(0) /= m32 | D8 /6 mem |
| `FDIV m64` | ST(0) /= m64 | DC /6 mem — **Note: DC /7 = FDIVR** |
| `FDIV ST(0), ST(i)` | ST(0) /= ST(i) | D8 F0+i |
| `FDIV ST(i), ST(0)` | ST(i) /= ST(0) | DC F8+i |
| `FDIVP ST(i), ST(0)` | ST(i) /= ST(0), pop | DE F8+i |

**Same reversal as FSUB**: DC/DE group reg=6 is FDIVR, reg=7 is FDIV.

#### FDIVR — Divide Reversed

`ST(0) = src / ST(0)`.

| Mnemonic | Description | Encoding |
|----------|-------------|----------|
| `FDIVR m32` | ST(0) = m32 / ST(0) | D8 /7 mem |
| `FDIVR m64` | ST(0) = m64 / ST(0) | DC /7 mem |
| `FDIVRP ST(i), ST(0)` | ST(i) = ST(0) / ST(i), pop | DE F0+i |

#### FIADD, FISUB, FIMUL, FIDIV — Integer Arithmetic

Converts memory integer to extended real, performs operation on ST(0), result in ST(0). No pop.

| Group | Word (16-bit) | Dword (32-bit) |
|-------|---------------|----------------|
| Add | `FIADD m16i` (DE /0) | `FIADD m32i` (DA /0) |
| Sub | `FISUB m16i` (DE /4) | `FISUB m32i` (DA /4) |
| Subr | `FISUBR m16i` (DE /5) | `FISUBR m32i` (DA /5) |
| Mul | `FIMUL m16i` (DE /1) | `FIMUL m32i` (DA /1) |
| Div | `FIDIV m16i` (DE /6) | `FIDIV m32i` (DA /6) |
| Divr | `FIDIVR m16i` (DE /7) | `FIDIVR m32i` (DA /7) |

#### FABS — Absolute Value

`ST(0) = |ST(0)|`. Clears the sign bit.

```
Encoding: D9 E1
```

#### FCHS — Change Sign

`ST(0) = -ST(0)`. Toggles the sign bit.

```
Encoding: D9 E0
```

#### FSQRT — Square Root

`ST(0) = sqrt(ST(0))`. Raises IE if ST(0) < 0.

```
Encoding: D9 FA
```

#### FRNDINT — Round to Integer

Round `ST(0)` to an integer value using the current rounding mode (RC field of CW). The result remains a floating-point type.

```
Encoding: D9 FC
```

#### FSCALE — Scale by Power of 2

`ST(0) = ST(0) × 2^⌊ST(1)⌋`. ST(1) is **not** popped.

```
Encoding: D9 FD
```

Used to efficiently multiply or divide by powers of 2.

#### FXTRACT — Extract Exponent and Significand

Splits `ST(0)` into exponent and significand:
- Pushes the biased exponent as a floating-point value onto the stack (becomes ST(0))
- The original `ST(0)` is replaced by the significand in the range [1.0, 2.0)

After FXTRACT: `ST(0)` = exponent, `ST(1)` = significand.

```
Encoding: D9 F4
```

#### FPREM — Partial Remainder

Computes `ST(0) mod ST(1)` (integer modulo) and stores the result in `ST(0)`. ST(1) is **not** popped. Sets C2=1 if the result is partial (further reductions needed), C2=0 when complete.

Note: FPREM uses truncation toward zero (C-style `fmod`), not IEEE 754 remainder.

```
Encoding: D9 F8
```

### 5.3 Comparison

#### FCOM — Compare Real

Compare `ST(0)` with operand; set condition codes C0, C2, C3 in status word. Neither operand is popped.

| Mnemonic | Description | Encoding |
|----------|-------------|----------|
| `FCOM m32` | Compare ST(0) with m32 | D8 /2 mem |
| `FCOM m64` | Compare ST(0) with m64 | DC /2 mem |
| `FCOM ST(i)` | Compare ST(0) with ST(i) | D8 D0+i |
| `FCOM` | Compare ST(0) with ST(1) | D8 D1 |

#### FCOMP — Compare Real and Pop

Same as FCOM but pops ST(0) afterwards.

| Mnemonic | Encoding |
|----------|----------|
| `FCOMP m32` | D8 /3 mem |
| `FCOMP m64` | DC /3 mem |
| `FCOMP ST(i)` | D8 D8+i |

#### FCOMPP — Compare Real and Pop Twice

Compare ST(0) with ST(1), set condition codes, pop twice.

```
Encoding: DE D9
```

#### FICOM — Compare Integer

Convert memory integer to extended real, compare with ST(0).

| Mnemonic | Encoding |
|----------|----------|
| `FICOM m16i` | DE /2 mem |
| `FICOM m32i` | DA /2 mem |
| `FICOMP m16i` | DE /3 mem (pop) |
| `FICOMP m32i` | DA /3 mem (pop) |

#### FTST — Test for Zero

Compare `ST(0)` with 0.0. Sets condition codes as if `FCOM +0.0` was performed.

```
Encoding: D9 E4
```

#### FXAM — Examine

Classify `ST(0)` and write the class into C0, C1, C2, C3 of the status word. See [Status Word](#23-status-word) table for the classification encoding.

```
Encoding: D9 E5
```

#### FSTSW — Store Status Word

Copy the status word to a memory word or (on 80287+) to AX.

| Mnemonic | Encoding |
|----------|----------|
| `FSTSW m16` | DD /7 mem |
| `FNSTSW m16` | DD /7 mem (no preceding FWAIT) |

**On the 8087, `FSTSW AX` is not available.** It was added with the 80287. On the 8087, the status word must be stored to memory and then loaded into AX with `MOV AX, [sw_mem]`.

### 5.4 Transcendental

These instructions are slow (100s of clocks) and assume their inputs are in the correct range. The programmer must reduce arguments to the required range before calling them.

#### F2XM1 — 2^x − 1

`ST(0) = 2^ST(0) − 1`. Input must be in the range **[−0.5, +0.5]**.

```
Encoding: D9 F0
```

To compute `2^x` for arbitrary `x`: decompose `x = integer + fraction` where `|fraction| ≤ 0.5`, compute `F2XM1(fraction) + 1`, then scale by `2^integer` using `FSCALE`.

#### FYL2X — y × log₂(x)

`ST(1) = ST(1) × log₂(ST(0))`, then pop. Requires `ST(0) > 0`.

```
Encoding: D9 F1
```

To compute `log₂(x)`: push 1.0, push x, then `FYL2X`. Result = `1.0 × log₂(x)`.  
To compute `ln(x)`: use `FYL2X` with y = `1/log₂(e)` = `ln(2)` ≈ 0.693147.  
To compute `log₁₀(x)`: use y = `1/log₂(10)` ≈ 0.30103.

#### FYL2XP1 — y × log₂(x + 1)

`ST(1) = ST(1) × log₂(ST(0) + 1)`, then pop. More accurate than `FYL2X` when `ST(0)` is close to zero. Input `ST(0)` must satisfy `|ST(0)| < 1 − √2/2 ≈ 0.2929`.

```
Encoding: D9 F9
```

#### FPTAN — Partial Tangent

Computes `tan(ST(0))`. The argument must be in the range `[0, π/4]`. After execution:
- `ST(0)` is replaced by `tan(x)` / a partial-tangent result Y
- `1.0` is pushed onto the stack so that `ST(0) = 1.0` and `ST(1) = tan(x)`

Divide `ST(1)` by `ST(0)` to obtain `tan(x)` in a single floating-point value. Argument reduction to `[0, π/4]` using `FPREM` and symmetry must be performed by the calling code.

```
Encoding: D9 F2
```

#### FPATAN — Partial Arctangent

`ST(1) = arctan(ST(1) / ST(0))`, then pop. Result is in `(−π, +π]`. Does not require argument reduction; inputs can be any finite values.

```
Encoding: D9 F3
```

To compute `arctan(x)`: push x, push 1.0, then `FPATAN`. Result = `arctan(x/1.0)`.

### 5.5 Constant Loading

These instructions push commonly used constants onto the stack, computed to full extended-precision accuracy.

| Mnemonic | Value pushed | Encoding |
|----------|-------------|----------|
| `FLDZ` | +0.0 | D9 EE |
| `FLD1` | +1.0 | D9 E8 |
| `FLDPI` | π ≈ 3.14159265358979... | D9 EB |
| `FLDL2E` | log₂(e) ≈ 1.44269504... | D9 EA |
| `FLDL2T` | log₂(10) ≈ 3.32192809... | D9 E9 |
| `FLDLG2` | log₁₀(2) ≈ 0.30103000... | D9 EC |
| `FLDLN2` | ln(2) ≈ 0.69314718... | D9 ED |

```asm
; Example: compute sin(x) via FPTAN/FPATAN
; sin(x) = 2*tan(x/2) / (1 + tan²(x/2))   for small x
; Better: use the identity cos²+sin²=1 with FPTAN
        FLD     x           ; ST(0) = x
        FPTAN               ; ST(0) = 1.0, ST(1) = tan(x)  [for x in 0..π/4]
        FXCH                ; ST(0) = tan(x), ST(1) = 1.0
        FPATAN              ; arctan(tan(x)/1.0) = x  [not useful here]
        ; typically use: sin = tan/sqrt(1+tan²), cos = 1/sqrt(1+tan²)
```

### 5.6 Control

#### FINIT / FNINIT — Initialize FPU

Reset the FPU to its default state: control word `0x037F`, status word 0, all registers tagged Empty.

- `FINIT` — preceded by `FWAIT` (synchronises with CPU)
- `FNINIT` — no preceding `FWAIT` (used during detection, or when no synchronisation is needed)

```
FINIT  encoding: 9B DB E3
FNINIT encoding: DB E3
```

#### FWAIT — Wait for FPU

`9B` — causes the CPU to wait until the FPU completes its current instruction. Place `FWAIT` before any instruction that reads a result written by the FPU, and before any FPU instruction that follows a CPU instruction that modifies memory used by the FPU.

#### FLDCW — Load Control Word

Load a 16-bit value from memory into the control word.

```
Encoding: D9 /5 mem
```

```asm
        FLDCW   [new_control_word]
```

#### FSTCW / FNSTCW — Store Control Word

Copy the control word to a 16-bit memory location.

- `FSTCW` — with preceding `FWAIT`
- `FNSTCW` — without `FWAIT`

```
Encoding: D9 /7 mem
```

#### FSTSW / FNSTSW — Store Status Word

Copy the status word to a 16-bit memory location.

```
Encoding: DD /7 mem
```

#### FSAVE / FNSAVE — Save FPU State

Save the complete 108-byte FPU state (control word, status word, tag word, instruction pointer, data pointer, and all 8 registers) to memory. Implicitly executes `FNINIT` after saving.

```
Encoding: DD /6 mem
```

#### FRSTOR — Restore FPU State

Restore FPU state from the 108-byte block previously written by `FSAVE`.

```
Encoding: DD /4 mem
```

#### FSTENV / FNSTENV — Store Environment

Save the 14-byte FPU environment (control, status, tag words and pointers) without saving register values.

```
Encoding: D9 /6 mem
```

Layout of the 14-byte environment block (real mode):
```
Offset  0    Control word (2 bytes)
Offset  2    Status word (2 bytes)
Offset  4    Tag word (2 bytes)
Offset  6    Instruction pointer offset (2 bytes)
Offset  8    CS selector | opcode bits (2 bytes)
Offset 10    Operand pointer offset (2 bytes)
Offset 12    Operand segment (2 bytes)
```

#### FLDENV — Load Environment

Restore the 14-byte environment block written by `FSTENV`.

```
Encoding: D9 /4 mem
```

#### FNCLEX — Clear Exception Flags

Clear the exception flags in the status word (bits 0–7) without `FWAIT`. Useful in exception handlers.

```
Encoding: DB E2
```

#### FNOP — No Operation

```
Encoding: D9 D0
```

#### FDECSTP — Decrement Stack Pointer

Decrement TOP by 1 (rotate the stack pointer without moving data). The new `ST(0)` retains whatever value and tag it previously had. Used for low-level stack manipulation.

```
Encoding: D9 F6
```

#### FINCSTP — Increment Stack Pointer

Increment TOP by 1. Marks the old `ST(0)` as Empty.

```
Encoding: D9 F7
```

#### FFREE — Free Register

Mark a register as Empty without changing TOP.

```
Encoding: DD C0+i     (8087: reg=0 = FFREE ST(i))
```

## 6. Operand Encoding

8087 instructions use a ModRM byte identical in structure to the 8086 ModRM byte. The opcode byte selects the instruction group (`D8`–`DF`), and the ModRM byte's `reg` field selects the operation within the group, while `mod` and `rm` encode the memory address or register operand.

**ModRM byte:**
```
Bits 7–6   mod   00=mem, no disp; 01=mem+disp8; 10=mem+disp16; 11=register
Bits 5–3   reg   operation selector within the D8–DF group
Bits 2–0   rm    base register or ST(i) index when mod=11
```

When `mod=11` (register form), `rm` is the stack register index (0–7) relative to TOP.

## 7. Exception Handling

The 8087 can signal six exception types. Each has a mask bit in the control word and a sticky flag bit in the status word.

| Exception | Mask bit | Flag bit | Typical cause |
|-----------|----------|----------|---------------|
| Invalid Operation (IE) | IM (bit 0) | IE (bit 0) | NaN operand, stack overflow/underflow, 0/0, ∞−∞ |
| Denormal Operand (DE) | DM (bit 1) | DE (bit 1) | Denormalised (gradual underflow) input |
| Zero Divide (ZE) | ZM (bit 2) | ZE (bit 2) | Division by zero with finite nonzero dividend |
| Overflow (OE) | OM (bit 3) | OE (bit 3) | Result too large to represent |
| Underflow (UE) | UM (bit 4) | UE (bit 4) | Result too small (before rounding) |
| Precision (PE) | PM (bit 5) | PE (bit 5) | Inexact result (very common, usually left masked) |

When an exception occurs and its mask bit is **set** (masked), the 8087 produces a default result (e.g. quiet NaN for IE, ∞ for ZE) and sets the flag bit. Execution continues.

When the mask bit is **clear** (unmasked), the 8087 asserts the `ERROR#` pin. On a PC this triggers **INT 2** (NMI). The OS or application must install an NMI handler to recover. Most applications leave all exceptions masked.

**Default control word `0x037F`** has all six exception bits set (masked), extended precision, round-to-nearest. This is the safe default for most applications.

### Checking for exceptions after a sequence

```asm
        ; ... FPU operations ...
        FWAIT                       ; ensure FPU is done
        FNSTSW  [sw_temp]           ; read status word without FWAIT
        MOV     AX, [sw_temp]
        TEST    AX, 0x000D          ; check IE | ZE | OE (bits 0,2,3)
        JNZ     handle_error
```

### Clearing exception flags

```asm
        FNCLEX                      ; DB E2 — clears exception flags in status word
```

## 8. Programming Techniques

### 8.1 Stack Discipline

The 8087 register stack holds exactly 8 values. Overflowing (pushing onto a full stack) or underflowing (popping from an empty stack) causes an Invalid Operation exception and silently corrupts results when exceptions are masked.

**Rules:**
1. Every `FLD`/`FILD`/`FBLD`/`FLDCONSTANT` pushes one value — ensure a free slot exists.
2. Every `FSTP`/`FISTP`/`FBSTP` pops one value.
3. Every `P`-suffix instruction (`FADDP`, `FMULP`, etc.) pops one value after the operation.
4. `FCOMPP` pops twice.
5. `FYL2X` and `FYL2XP1` pop once (consume both ST(0) and ST(1), leave result in ST(0)).
6. `FPTAN` pushes one extra value (pushes 1.0 after computation).
7. On function entry and exit, the stack must be in the same state as the calling convention expects (usually empty, or with a return value in ST(0)).

### 8.2 Integer Conversion (Trunc and Round)

The control word's **RC field** determines how `FRNDINT`, `FIST`, and `FISTP` round floating-point values to integers.

**Truncation (as in C cast `(int)x`, Pascal `Trunc`):**

```asm
        ; Save current control word, set truncation mode, convert, restore
        FSTCW   [old_cw]            ; save CW
        FWAIT
        MOV     AX, [old_cw]
        OR      AX, 0x0C00          ; set RC = 11 (truncate)
        MOV     [new_cw], AX
        FLDCW   [new_cw]            ; load truncation CW
        FISTP   DWORD PTR [result]  ; store integer (truncated)
        FLDCW   [old_cw]            ; restore CW
```

**Round to nearest (Pascal `Round`, round half away from zero):**

Pascal's `Round()` function rounds 0.5 **away from zero** (e.g. Round(1.5)=2, Round(2.5)=3). The 8087's default RC=00 is **round-half-to-even** (IEEE 754 banker's rounding: Round(1.5)=2, Round(2.5)=2). For exact half-integer values these differ.

TP5's common implementation of `Round()`: load the value, set RC=00 (nearest-even), call FISTP. For most values these agree; only exact half-integers on even boundaries differ.

**Floor and ceiling:**

```asm
        ; Floor: set RC=01
        MOV     AX, [old_cw]
        AND     AX, 0xF3FF
        OR      AX, 0x0400          ; RC = 01 (floor)
        MOV     [new_cw], AX
        FLDCW   [new_cw]

        ; Ceiling: set RC=10
        AND     AX, 0xF3FF
        OR      AX, 0x0800          ; RC = 10 (ceiling)
```

### 8.3 Transcendental Functions

The 8087 provides only `F2XM1`, `FYL2X`, `FYL2XP1`, `FPTAN`, `FPATAN`, and `FSCALE`. All other transcendental functions must be constructed from these.

**exp(x) — base-e exponential:**

Use the identity `e^x = 2^(x × log₂(e))`.

```asm
        ; Input: ST(0) = x
        FLDL2E                  ; ST(0)=log2(e), ST(1)=x
        FMULP   ST(1), ST(0)    ; ST(0) = x * log2(e)
        ; Now compute 2^ST(0):
        FLD     ST(0)           ; ST(0)=y, ST(1)=y  (y = x*log2e)
        FRNDINT                 ; ST(0)=trunc(y), ST(1)=y
        FSUB    ST(1), ST(0)    ; ST(1)=frac(y), ST(0)=trunc(y)
        FXCH                    ; ST(0)=frac(y), ST(1)=trunc(y)
        F2XM1                   ; ST(0)=2^frac(y)-1
        FLD1
        FADD                    ; ST(0)=2^frac(y), ST(1)=trunc(y)
        FSCALE                  ; ST(0)=2^frac(y)*2^trunc(y)=2^y=e^x
        FXCH    ST(1)
        FSTP    ST(0)           ; pop the trunc(y) we no longer need
```

**ln(x) — natural logarithm:**

Use `ln(x) = FYL2X(x, ln(2))`.

```asm
        ; Input: ST(0) = x
        FLDLN2                  ; ST(0)=ln(2), ST(1)=x
        FXCH                    ; ST(0)=x, ST(1)=ln(2)
        FYL2X                   ; ST(0)=ln(2)*log2(x)=ln(x)
```

**log₁₀(x):**

```asm
        FLDLG2                  ; ST(0)=log10(2)
        FXCH
        FYL2X                   ; ST(0)=log10(2)*log2(x)=log10(x)
```

**sin(x) and cos(x):**

Compute tan(x/2) using `FPTAN`, then derive sin and cos via double-angle formulae, or use:

```asm
        ; sin(x) via FPTAN + identity: sin = 2t/(1+t²), t = tan(x/2)
        ; For argument-reduced x in [-π/4, π/4]:
        FLD     x               ; ST(0)=x
        FPTAN                   ; ST(0)=1.0, ST(1)=tan(x)  (for |x|≤π/4)
        ; cos(x) = 1/sqrt(1 + tan²(x)):
        FLD     ST(0)           ; dup 1.0
        FLD     ST(2)           ; dup tan(x)
        FMUL    ST(0), ST(0)    ; tan²(x)
        FADD    ST(0), ST(2)    ; 1 + tan²(x)
        FSQRT                   ; sqrt(1+tan²(x))
        FDIVR   ST(0), ST(1)    ; 1/sqrt(1+tan²(x)) = cos(x)
        ; etc.
```

In practice, use a runtime library routine (BFLOAT, Pascal RTL) rather than inlining these sequences manually.

**arctan(x):**

`FPATAN` computes `arctan(y/x)` for any finite y, x.

```asm
        FLD     x               ; ST(0) = x
        FLD1                    ; ST(0) = 1.0, ST(1) = x
        FPATAN                  ; ST(0) = arctan(x/1.0) = arctan(x)
```

**sqrt(x):**

Use `FSQRT` directly; no argument reduction needed.

```asm
        FLD     x
        FSQRT                   ; ST(0) = sqrt(x)
```

### 8.4 Comparison and Branching

The 8087 comparison instructions set C0, C2, C3 in the status word. These bits do **not** map directly to 8086 CPU flags. The programmer must transfer the status word to a CPU register and test the relevant bits.

**Standard comparison sequence (8087, real mode):**

```asm
        FLD     a
        FCOMP   b               ; compare a with b, pop
        FWAIT                   ; wait for result
        FNSTSW  [sw_temp]       ; store status word to memory
        MOV     AX, [sw_temp]
        SAHF                    ; transfer AH to CPU FLAGS: C0→CF, C2→PF, C3→ZF
        ; Now use 8086 conditional jumps:
        JB      label_lt        ; jump if a < b  (CF=1, i.e. C0=1)
        JE      label_eq        ; jump if a = b  (ZF=1, i.e. C3=1)
        JA      label_gt        ; jump if a > b  (CF=0, ZF=0)
        JP      label_unord     ; jump if unordered (PF=1, i.e. C2=1)
```

The `SAHF` instruction loads AH into the lower byte of FLAGS:
- AH bit 0 → CF (Carry)
- AH bit 2 → PF (Parity)
- AH bit 6 → ZF (Zero)
- AH bit 7 → SF (Sign)

Status word AH after `MOV AX, [sw_temp]; MOV AH, AH` — actually `SAHF` uses the high byte of AX after `FNSTSW [mem]; MOV AX, [mem]`:

```
Status bit: C3(bit14) C2(bit10) C0(bit8)
In AH:      C3=bit6   C2=bit2   C0=bit0
After SAHF: ZF=C3     PF=C2     CF=C0
```

**On 80287 and later**, `FNSTSW AX` eliminates the memory round-trip:

```asm
        FNSTSW  AX              ; 80287+: direct to AX (not available on 8087!)
        SAHF
        JB      less_than
```

**8087-compatible way** (always works):

```asm
        FNSTSW  [temp]
        MOV     AX, [temp]
        SAHF
```

## 9. Compiling for the 8087

### 9.1 Assembler (MASM / TASM)

Microsoft MASM and Borland TASM both support 8087 mnemonics. The key directives are:

**Processor directives:**

```asm
.8086           ; 8086 instructions only (default)
.8087           ; enable 8087 FPU instructions (used with .8086)
.286            ; enable 286 instructions
.287            ; enable 80287 FPU instructions
.386            ; enable 386 instructions
.387            ; enable 80387 FPU instructions (adds FSIN, FCOS, etc.)
```

To target a plain PC-XT:
```asm
.8086
.8087
```

**FWAIT generation:** MASM/TASM automatically prepend `FWAIT` (`9B`) before most FPU instructions when using `.8086/.8087`. The `FN`-prefixed variants (`FNINIT`, `FNCLEX`, `FNSTSW`, `FNSTCW`) suppress the automatic `FWAIT`.

**Memory operand sizes:** Always specify the operand size explicitly:

```asm
FILD    WORD PTR  [bx]          ; 16-bit integer
FILD    DWORD PTR [bx]          ; 32-bit integer  (DB group)
FILD    QWORD PTR [bx]          ; 64-bit integer  (DF group)
FLD     DWORD PTR [bx]          ; 32-bit single real
FLD     QWORD PTR [bx]          ; 64-bit double real
FLD     TBYTE PTR [bx]          ; 80-bit extended real
FBLD    TBYTE PTR [bx]          ; 80-bit packed BCD
```

**Minimal 8087 assembly program (TASM syntax):**

```asm
.MODEL  SMALL
.8086
.8087
.STACK  256
.DATA
        x       DD  3.14159     ; single-real constant
        result  DD  ?
.CODE
main    PROC    FAR
        MOV     AX, @DATA
        MOV     DS, AX
        FNINIT
        FLD     DWORD PTR [x]   ; ST(0) = 3.14159
        FMUL    ST(0), ST(0)    ; ST(0) = pi^2
        FSTP    DWORD PTR [result]
        FWAIT
        MOV     AX, 4C00h
        INT     21h
main    ENDP
END     main
```

**Compile and link:**
```
tasm    program.asm
tlink   program.obj
```

Or with MASM:
```
masm    program.asm;
link    program.obj;
```

### 9.2 Turbo Pascal

Turbo Pascal 4.0 and later support the 8087 via compiler directives.

**Key directives:**

| Directive | Meaning |
|-----------|---------|
| `{$N-}` | Use software floating-point emulation (default) |
| `{$N+}` | Use native 8087 instructions; abort if no FPU present |
| `{$E+}` | Enable Borland's software FPU emulator (intercepts INT 3Ch etc.) |
| `{$E-}` | Disable the software emulator |

For true 8087 code (no emulation), use `{$N+,E-}` at the top of the file.

**Float types and 8087 types:**

| Pascal type | 8087 type | Size |
|-------------|-----------|------|
| `Single` | Single real | 4 bytes |
| `Double` | Double real | 8 bytes |
| `Extended` | Extended real | 10 bytes |
| `Comp` | 64-bit integer (stored in FPU) | 8 bytes |
| `Real` | Borland 6-byte real (software only) | 6 bytes |

`Real` is a Borland-specific 6-byte format not supported natively by the 8087. It requires the software emulator (`{$N-}` mode) and cannot be used with `{$N+}`.

**Example source file:**

```pascal
{$N+,E-}
program FPUTest;

var
  x, y: Extended;

begin
  x := 2.0;
  y := Sqrt(x);                   { uses FSQRT }
  Writeln('sqrt(2) = ', y:0:15);
end.
```

**Compiling from command line (Turbo Pascal 5.x):**
```
tpc /N+ /E- fputest.pas
```

Or inside the IDE: set Options → Compiler → Numeric Processing to "8087/80287".

**Built-in transcendental functions** (`{$N+}` mode):

`Sin`, `Cos`, `ArcTan`, `Ln`, `Exp`, `Sqrt`, `Int`, `Frac`, `Abs`, `Sqr`, `Trunc`, `Round` all compile to FPU instruction sequences when `{$N+}` is active. The TP5 runtime library contains optimised inline FPU code for these.

**Detecting the 8087 at runtime in Pascal:**

```pascal
{$N-}   { must be in emulation mode to use this safely }
function Has8087: Boolean;
var
  cw: Word;
begin
  asm
    FNINIT
    FSTCW   cw
    FWAIT
  end;
  Has8087 := (cw AND $FF) = $7F;
end;
```

### 9.3 Turbo C / Borland C++

**Compiler switches:**

| Switch | Meaning |
|--------|---------|
| `-f` | Software FPU emulation (default in small/medium models) |
| `-f87` | Require 8087, emit native FPU instructions, no emulation |
| `-f287` | Require 80287 |
| `-f387` | Require 80387 |
| `-ff` | Fast floating-point (may reduce precision) |

```
tcc -f87 -ms program.c
```

**Float types:**

| C type | 8087 type |
|--------|-----------|
| `float` | Single real (32-bit) |
| `double` | Double real (64-bit) |
| `long double` | Extended real (80-bit) |

With `-f87`, all `double` arithmetic uses native 8087 instructions. Inline assembly is available with `asm { ... }` blocks.

**Inline assembly example:**

```c
#pragma inline
double fsqrt(double x) {
    asm {
        fld     QWORD PTR x
        fsqrt
        fstp    QWORD PTR x
    }
    return x;
}
```

**Pragma for FPU control word:**

```c
#include <float.h>

_control87(MCW_RC, RC_CHOP);    /* set truncation mode */
int n = (int) x;                 /* FISTP in truncation mode */
_control87(MCW_RC, RC_NEAR);    /* restore round-to-nearest */
```

### 9.4 Microsoft C

**Compiler switches:**

| Switch | Meaning |
|--------|---------|
| `/FPi` | IEEE emulation (calls runtime library, default) |
| `/FPi87` | Require 8087, native instructions |
| `/FPa` | Altmath library (fast but less accurate) |
| `/FPc` | Calls runtime library via interrupt (compatible emulation) |

```
cl /FPi87 /AS program.c
```

**Float types** are the same as Borland C: `float` = 32-bit, `double` = 64-bit, `long double` = 80-bit extended (MSC 5.x+).

## 10. Complete Examples

### Example 1: Quadratic formula in TASM

Compute the real roots of `ax² + bx + c = 0` using the 8087.

```asm
.MODEL  SMALL
.8086
.8087
.STACK  512
.DATA
        a       DD   1.0
        b       DD  -5.0
        c       DD   6.0
        root1   DD   ?
        root2   DD   ?
        temp    DD   ?
.CODE
quad    PROC    FAR
        MOV     AX, @DATA
        MOV     DS, AX
        FNINIT

        ; discriminant = b² - 4ac
        FLD     DWORD PTR [b]   ; ST(0) = b
        FMUL    ST(0), ST(0)    ; ST(0) = b²
        FLD     DWORD PTR [a]   ; ST(0) = a,  ST(1) = b²
        FLD     DWORD PTR [c]   ; ST(0) = c,  ST(1) = a,  ST(2) = b²
        FMULP   ST(1), ST(0)    ; ST(0) = a*c, ST(1) = b²
        FLD1
        FLD1
        FADDP   ST(1), ST(0)    ; ST(0) = 2.0
        FADDP   ST(1), ST(0)    ; ST(0) = 4.0  (note: 2+2=4)
        ; Actually simpler:
        ; FIMUL WORD PTR [four]
        FMULP   ST(1), ST(0)    ; ST(0) = 4*a*c (but we have ST(0)=4, ST(1)=a*c)
        ; Let's redo cleanly:
        ; ...
        ; root1 = (-b + sqrt(disc)) / (2a)
        ; root2 = (-b - sqrt(disc)) / (2a)

        ; Compute 2a
        FLD     DWORD PTR [a]   ; ST(0) = a
        FLD     ST(0)           ; ST(0) = a, ST(1) = a
        FADDP   ST(1), ST(0)    ; ST(0) = 2a

        ; Compute sqrt(b²-4ac) — assume discriminant ≥ 0
        ; (discriminant already in temp from above)
        FLD     DWORD PTR [temp]; ST(0) = disc, ST(1) = 2a
        FSQRT                   ; ST(0) = sqrt(disc), ST(1) = 2a

        FLD     DWORD PTR [b]   ; ST(0) = b, ST(1) = sqrt(disc), ST(2) = 2a
        FCHS                    ; ST(0) = -b

        ; root1 = (-b + sqrt) / 2a
        FLD     ST(0)           ; dup -b
        FADD    ST(0), ST(2)    ; ST(0) = -b + sqrt(disc)
        FDIV    ST(0), ST(4)    ; ST(0) = (-b+sqrt)/2a   [ST(4)=2a]
        FSTP    DWORD PTR [root1]

        ; root2 = (-b - sqrt) / 2a
        FSUB    ST(0), ST(1)    ; ST(0) = -b - sqrt(disc)
        FDIV    ST(0), ST(2)    ; ST(0) = (-b-sqrt)/2a
        FSTP    DWORD PTR [root2]

        FSTP    ST(0)           ; discard sqrt(disc)
        FSTP    ST(0)           ; discard 2a
        FWAIT

        MOV     AX, 4C00h
        INT     21h
quad    ENDP
END     quad
```

### Example 2: Exponentiation in Turbo Pascal

```pascal
{$N+,E-}
program PowerTest;

{ Compute x^y = 2^(y * log2(x)) using 8087 transcendentals }
function Power(x, y: Extended): Extended;
var
  t: Extended;
  ti: Integer;
begin
  if x <= 0.0 then
    RunError(207);    { undefined }
  { t = y * log2(x) }
  t := y * (Ln(x) / Ln(2));
  { 2^t = 2^int(t) * 2^frac(t) }
  ti := Trunc(t);
  t := t - ti;        { t is now fractional part, |t| < 1 }
  { F2XM1 requires |t| <= 0.5; use identity 2^t = 2^(t-1)*2 if t > 0.5 }
  if t > 0.5 then begin
    t := t - 1.0;
    Inc(ti);
  end;
  { 2^t - 1 via F2XM1, then +1, then scale by 2^ti via FSCALE }
  Power := (Exp(t * Ln(2)) ) * Exp(ti * Ln(2));
  { Simpler: just use Exp(y * Ln(x)) which TP5 compiles to FPU ops }
end;

begin
  Writeln(Power(2.0, 10.0):0:0);    { 1024 }
  Writeln(Power(3.0, 3.0):0:0);     { 27 }
  Writeln(Exp(10.0 * Ln(2.0)):0:6); { same result via Exp/Ln }
end.
```

### Example 3: FPU detection + safe sqrt in C

```c
/* Compile: tcc -f87 -ms detect.c */
#include <stdio.h>
#include <math.h>

static int fpu_detected = -1;   /* -1 = not yet checked */

int has_fpu(void) {
    unsigned int cw;
    if (fpu_detected >= 0) return fpu_detected;
    _asm {
        fninit
        fstcw   cw
        fwait
    }
    fpu_detected = ((cw & 0x00FF) == 0x007F) ? 1 : 0;
    return fpu_detected;
}

double safe_sqrt(double x) {
    if (has_fpu()) {
        double result;
        _asm {
            fld     QWORD PTR x
            fsqrt
            fstp    QWORD PTR result
            fwait
        }
        return result;
    } else {
        return sqrt(x);    /* software fallback */
    }
}

int main(void) {
    printf("FPU present: %s\n", has_fpu() ? "yes" : "no");
    printf("sqrt(2) = %.15f\n", safe_sqrt(2.0));
    return 0;
}
```

| D9 | FLD m32 | — | FST m32 | FSTP m32 | FLDENV | FLDCW | FSTENV | FSTCW |
| DA | FIADD m32i | FIMUL m32i | FICOM m32i | FICOMP m32i | FISUB m32i | FISUBR m32i | FIDIV m32i | FIDIVR m32i |
| DB | FILD m32i | — | FIST m32i | FISTP m32i | — | FLD m80 | — | FSTP m80 |
| DC | FADD m64 | FMUL m64 | FCOM m64 | FCOMP m64 | FSUB m64 | FSUBR m64 | FDIV m64 | FDIVR m64 |
| DD | FLD m64 | — | FST m64 | FSTP m64 | FRSTOR | — | FSAVE | FSTSW |

### Register forms (mod = 11)

| Opcode | reg=0 | reg=1 | reg=2 | reg=3 | reg=4 | reg=5 | reg=6 | reg=7 |
|--------|-------|-------|-------|-------|-------|-------|-------|-------|
| D8 | FADD ST,ST(i) | FMUL ST,ST(i) | FCOM ST(i) | FCOMP ST(i) | FSUB ST,ST(i) | FSUBR ST,ST(i) | FDIV ST,ST(i) | FDIVR ST,ST(i) |
| D9 | FLD ST(i) | FXCH ST(i) | FNOP¹ | — | see §5.6 table | see §5.5 table | see §5.4 table | see §5.4 table |
| DA | — | — | — | — | — | FUCOMPP² | — | — |
| DB | — | — | — | — | FNCLEX³ FNINIT⁴ | — | — | — |
| DC | FADD ST(i),ST | FMUL ST(i),ST | — | — | FSUBR ST(i),ST | FSUB ST(i),ST | FDIVR ST(i),ST | FDIV ST(i),ST |
| DD | FFREE ST(i) | — | FST ST(i) | FSTP ST(i) | — | — | — | — |
| DE | FADDP ST(i),ST | FMULP ST(i),ST | — | FCOMPP⁵ | FSUBRP ST(i),ST | FSUBP ST(i),ST | FDIVRP ST(i),ST | FDIVP ST(i),ST |
| DF | — | — | — | — | — | — | — | — |

Notes:
1. FNOP: only rm=0; D9 D0
2. FUCOMPP: only DA ED (reg=5, rm=1); pops twice
3. FNCLEX: only DB E2 (reg=4, rm=2)
4. FNINIT: only DB E3 (reg=4, rm=3)
5. FCOMPP: only DE D9 (reg=3, rm=1)

### FWAIT prefix summary

| Form | Prefix | Notes |
|------|--------|-------|
| `FINIT` | `9B DB E3` | With FWAIT — use for normal initialisation |
| `FNINIT` | `DB E3` | Without FWAIT — use during detection |
| `FCLEX` | `9B DB E2` | With FWAIT |
| `FNCLEX` | `DB E2` | Without FWAIT |
| `FSTSW` | `9B DD /7` | With FWAIT |
| `FNSTSW` | `DD /7` | Without FWAIT — safe for status-word polling loops |
| `FSTCW` | `9B D9 /7` | With FWAIT |

| `FSTENV` | `9B D9 /6` | With FWAIT |

| `FSAVE` | `9B DD /6` | With FWAIT |
| `FNSAVE` | `DD /6` | Without FWAIT |
